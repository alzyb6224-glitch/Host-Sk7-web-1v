# 🚀 SK7 Hosting — منصة الاستضافة المتقدمة

منصة استضافة حقيقية كاملة تعمل على أي VPS — **بدون Docker**. كل مشروع (موقع / API / بوت تيليجرام) يشتغل كـ عملية نظام حقيقية (process)، معزولة بمجلدها الخاص وبيئة Python افتراضية (venv) خاصة به، ومراقَبة تلقائياً بحيث تعيد نفسها إذا وقفت — يعني تشغيل 24/7 حقيقي.

---

## 📁 هيكل الملفات

```
sk7hosting/
├── app.py              ← الخادم الرئيسي (Flask) + محرك التشغيل الحقيقي
├── requirements.txt    ← مكتبات Python للوحة نفسها
├── install.sh          ← تثبيت تلقائي كامل على VPS
├── templates/
│   └── index.html      ← الواجهة الأمامية الكاملة
├── data/                ← قاعدة البيانات (JSON)
├── uploads/             ← الملفات المرفوعة
└── projects/            ← مشاريع المستخدمين (كل مشروع بمجلد + venv خاص به)
```

## ⚡ التثبيت على VPS (أسرع طريقة)

```bash
# 1. انسخ الملفات إلى الـ VPS
scp -r sk7hosting/ root@YOUR_VPS_IP:/opt/

# 2. اتصل بالـ VPS
ssh root@YOUR_VPS_IP

# 3. شغّل سكريبت التثبيت
cd /opt/sk7hosting
chmod +x install.sh
bash install.sh
```

سيقوم السكريبت تلقائياً بـ:
- تحديث النظام
- تثبيت Python (مع venv) + Node.js + PHP — بدون Docker
- تثبيت Nginx كـ Reverse Proxy
- إعداد Firewall (UFW)
- إعداد Fail2Ban (حماية ضد الاختراق)
- إنشاء Systemd Service (يبدأ تلقائياً عند إعادة تشغيل السيرفر، ويشغّل أي مشروع كان شغالاً قبل الإعادة)
- إعداد نسخ احتياطي يومي تلقائي

## ⚙️ كيف يشتغل التشغيل بدون Docker؟

- **اكتشاف تلقائي كامل**: ارفع ملفاتك بدون ما تحدد شي، واللوحة تكتشف تلقائياً نوع المشروع (Python/Node/Go/PHP/Static)، والفريمورك (Django مثلاً)، وملف التشغيل الرئيسي — تماماً متل Railway/Render. تقدر دايماً تتجاوز الاكتشاف وتحدد يدوياً لو حبيت.
- **Python**: venv خاص بالمشروع، تثبيت `requirements.txt` تلقائياً. **لو ما فيه requirements.txt أصلاً**، تفحص اللوحة الاستيرادات بالكود (`import flask`، `import requests`...) وتخمّن المكتبات المطلوبة من قائمة مكتبات شائعة (Flask, FastAPI, Django, python-telegram-bot, aiogram, discord.py, pymongo, psycopg2, redis...) وتولّد `requirements.txt` تلقائياً قبل التثبيت.
- **Django**: يُكتشف تلقائياً (وجود `manage.py`)، يشغّل `migrate` تلقائياً، ثم `manage.py runserver`. **ملاحظة**: لازم `ALLOWED_HOSTS` بملف `settings.py` يسمح بالدومين المستخدم (أو `['*']` للتجربة).
- **Node**: يقرأ `package.json` (حقل `main` أو `scripts.start`) لتحديد ملف التشغيل، ويشغّل `npm install` تلقائياً.
- **Go**: يُكتشف تلقائياً (وجود `go.mod`)، يبني المشروع بـ `go build` ثم يشغّل الملف التنفيذي مباشرة. **يحتاج Go مثبت على السيرفر** (`install.sh` يثبته تلقائياً كخطوة اختيارية).
- **PHP**: يشتغل عبر السيرفر المدمج في PHP (`php -S`)، مع `composer install` تلقائي لو فيه `composer.json`.
- **Static**: يشتغل عبر سيرفر ملفات بسيط على بورته الخاص.
- **بوتات تيليجرام/ديسكورد**: أي سكربت بايثون أو Node طويل الأمد (polling/gateway loop) يشتغل بنفس الآلية العادية — بدون حاجة لبورت أصلاً، والمراقب التلقائي (watchdog) يعيد تشغيله إذا كرشّ، فيبقى شغال 24/7.
- **Environment Variables**: تُمرَّر كمتغيرات بيئة حقيقية للعملية، **وتُكتب أيضاً بملف `.env`** داخل مجلد المشروع — فلو مشروعك يستخدم `python-dotenv` أو مكتبة `.env` مشابهة، بتشتغل تلقائياً بدون أي تعديل بكودك.
- **مراقب تلقائي (Watchdog)**: خيط خلفي يفحص كل المشاريع كل 5 ثواني، وإذا وقفت عملية بشكل غير متوقع يعيد تشغيلها فوراً (مع حماية من التكرار اللانهائي عند تعطّل حقيقي بالكود).
- **الاستمرارية بعد إعادة تشغيل اللوحة نفسها**: أي مشروع كان شغالاً يُعاد تشغيله تلقائياً عند إقلاع اللوحة من جديد (systemd أو إعادة تشغيل VPS).
- **أمر تشغيل مخصص**: لو احتجت تحكم كامل (مثلاً مشروع فيه أوامر build خاصة)، فيه حقل "أمر تشغيل مخصص" بصفحة النشر يتجاوز الإعداد الافتراضي بالكامل.

### 📁 إدارة الملفات والتيرمنل

كل مشروع فيه File Manager حقيقي من لوحة التحكم مباشرة: تصفح مجلدات المشروع، افتح أي ملف نصي وعدّله واحفظه، بدون SSH. والتيرمنل يشغّل أوامر فعلية داخل مجلد المشروع (وبيئته الافتراضية إن كان Python)، مع زر تثبيت مكتبة سريع (`pip install` / `npm install`) يعيد تشغيل المشروع تلقائياً بعد التثبيت.

### 🚦 حدود الموارد وعدد المشاريع

- كل عملية مشروع مقيّدة بحد أقصى للذاكرة (768MB افتراضياً، قابل للتعديل عبر متغير `SK7_PROJECT_MEM_MB`) وعدد العمليات الفرعية — شبكة أمان تمنع مشروع واحد من إسقاط السيرفر كامل.
- كل مستخدم عادي محدود بـ 8 مشاريع شغالة بنفس الوقت كحد أقصى (`MAX_RUNNING_PER_USER` بالكود) — الأدمن والـ DEV بدون حد.

### 🔗 الروابط — لماذا كانت تنكسر خلف Cloudflare Tunnel، وكيف انحلت

كل مشروع يشتغل داخلياً على بورت خاص فيه (9000-9999) على نفس الـ VPS. سابقاً كانت اللوحة تولّد الرابط بشكل `hostname:PORT` — وهذا يشتغل فقط لو كنت توصل للـ VPS مباشرة. أما خلف أي reverse proxy أو Cloudflare Tunnel (زي `trycloudflare.com`)، فالنفق يوجّه فقط لبورت واحد؛ فإضافة `:9002` على دومين النفق ما كانت توصل لأي شي — هذا كان سبب الرابط "المكسور".

**الحل**: أضفنا **Reverse Proxy حقيقي داخل اللوحة نفسها** على المسار `/app/<project_id>/`. يعني رابط مشروعك صار مثلاً:
```
https://your-tunnel-domain.trycloudflare.com/app/2d94f60e/
```
وهذا يشتغل عبر أي نفق أو reverse proxy لأنه كله يمر من نفس البورت اللي اللوحة شغالة عليه (لا حاجة لفتح بورت إضافي بالنفق أو الفايروول لكل مشروع). اللوحة كمان **تتحقق فعلياً** (HTTP probe حقيقي على `127.0.0.1:PORT`) قبل ما تعرض الرابط كـ "شغال" — فما راح تشوف رابط أخضر إلا وهو فعلاً يستجيب.

> ⚠️ **حدود الـ Proxy الحالي**: يدعم HTTP/HTTPS العادي (مواقع، APIs، REST) لكنه لا يدعم WebSockets أو streaming طويل الأمد حالياً. لو مشروعك يحتاج WebSocket (مثلاً socket.io)، استخدم بورته المباشر `VPS_IP:PORT` بدل مسار `/app/`.

> ⚠️ **ملاحظة أمان مهمة**: بدون Docker ما فيه عزل حقيقي بين المشاريع (كلها تشتغل بنفس المستخدم على نفس السيرفر). أضفنا حدود موارد أساسية لكل عملية (حد أقصى للذاكرة والعمليات الفرعية عبر `resource.setrlimit`) كشبكة أمان تمنع مشروع واحد من إسقاط السيرفر كامل، وحظرنا أخطر أوامر التيرمنل (حذف جذر النظام، fork bombs، إلخ)، لكن هذا **ليس عزل كامل** متل الحاويات. لو بتفتح الاستضافة لمستخدمين غير موثوقين، ننصح بتشغيل اللوحة داخل مستخدم Linux محدود الصلاحيات (غير root).

> ⚠️ **قواعد البيانات**: ما فيه توفير تلقائي لقواعد بيانات مُدارة (Postgres/MySQL/Redis كخدمة). المدعوم حالياً: SQLite يشتغل مباشرة (مجرد ملف داخل مجلد المشروع)، وأي قاعدة بيانات خارجية (Supabase, Neon, Railway DB, إلخ) توصلها عبر Environment Variables بشكل طبيعي. لو احتجت Postgres/Redis محلي على نفس الـ VPS، تقدر تثبته يدوياً (`apt install postgresql redis-server`) وتربط مشاريعك عليه عبر `localhost`.
